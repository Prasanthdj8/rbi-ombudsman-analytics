# Complaint Classifier

A text classifier that tags complaint-style text into RBI's published Ombudsman
complaint categories: Loans and Advances, Credit Cards, ATM/Debit Cards,
Mobile/Electronic Banking, Deposit Accounts, and Pension Related.

## Data provenance, read this first

RBI does not publish individual complaint narratives publicly. Everything RBI
publishes about complaints is aggregate (counts by category, by year, by
region), which is what the dashboard in `../dashboard/` uses.

The text used to train this classifier is entirely synthetic, written by the
project author to reflect realistic complaint phrasing within each of RBI's
real published categories. It is generated by `generate_synthetic_complaints.py`
and is not, and should not be read as, real complainant data. This is stated
here and in the project's top level README so there is no ambiguity about
what is real RBI data (the categories and the trend figures) and what is a
demonstration (the complaint text itself).

## Why this approach

Building a classifier without any text data to train on isn't possible, and
RBI's complaint narratives are confidential, which is appropriate given they
involve individual financial grievances. Generating clearly labeled synthetic
text in the real category structure is the honest way to demonstrate the
classification pipeline (the kind of system the CEP01 job profile asks for,
under "data mining and analytics" and "systematic classification of
complaints") without overstating what data was actually available.

## Pipeline

TF-IDF vectorization (unigrams and bigrams) feeding a linear SVM classifier.
This is a deliberately lightweight, fast, and interpretable choice, suited to
an internal MIS tool that needs to run quickly on a data dump rather than
needing the heaviest possible model. A transformer-based classifier (e.g.
fine-tuning a small BERT variant) would be a reasonable next step for a
production system handling more nuanced or ambiguous complaint text.

## A note on the accuracy number

Running `classify_complaints.py` will show very high accuracy (close to
100%) on the held-out test set. This is expected and is a property of the
synthetic dataset, not a claim about real-world performance. The synthetic
sentences within each category share fairly distinct vocabulary by
construction, which makes them easy to separate. Real complaint text would
be messier: complaints often reference multiple issues at once, use
inconsistent terminology, and require handling class imbalance (since real
complaint volumes are heavily skewed, with Loans and Advances and Credit
Cards far more common than Pension Related, as the actual RBI data in
`../data/` shows). The honest summary is: this demonstrates a working,
correctly structured pipeline, not a validated production accuracy figure.

## Running it

```bash
pip install scikit-learn pandas joblib
python generate_synthetic_complaints.py   # creates synthetic_complaints.csv
python classify_complaints.py             # trains, evaluates, saves the model
```

Outputs:
- `complaint_classifier.joblib` — the trained pipeline
- `metrics.json` — full classification report
- Console output showing predictions on new, unseen example sentences

## Files

| File | Purpose |
|---|---|
| `generate_synthetic_complaints.py` | Generates the labeled synthetic dataset |
| `synthetic_complaints.csv` | The generated dataset (240 samples, 6 categories) |
| `classify_complaints.py` | Trains and evaluates the TF-IDF + SVM classifier |
| `complaint_classifier.joblib` | Saved trained model (generated on run) |
| `metrics.json` | Saved classification report (generated on run) |
